하민석 UIUX Test

1번) 완료
UI 캔버스에 스크립트를 넣고 인풋필드와 버튼을 가지게 해서
버튼이 눌렸을 때 AddListener로 인풋필드의 텍스트를 출력하게 했다.

2번) 완료
UI 캔버스에 스크립트를 넣고 버튼 2개를 가지게 해서
각 버튼이 눌렸을 때 자신을 비활성화하고 다른 버튼을 활성화시키게 하면서
On Off 콘솔 메시지를 출력시키게 했다.

3번) 완료
UI 캔버스에 스크립트를 넣고 슬라이더를 가지게 해서
슬라이더의 값이 바뀌었을 때 값을 출력하도록 AddListener 했다.

4번) 완료
스테이지 슬롯을 생성할 때 인포매니져가 갖고 있는 스테이지 인포를
참조하여 클리어한 스테이지, 오픈된 스테이지, 잠긴 스테이지를 구분했고
클리어한 스테이지 슬롯을 생성할 때는 스테이지 인포의 별 개수를 참조하여
별 오브젝트를 활성화시켰다.

5번) 완료
UI 캔버스에 스크립트를 넣고 버튼과 이미지, 텍스트를 가지게 해서
버튼이 눌리면 이미지의 fill amount를 초기화하고 코루틴을 실행시킨다.
코루틴에서는 스킬의 쿨타임을 진행시키면서
fill amount를 증가시키고, 남은 쿨타임 텍스트를 보여준다.

6번) 완료
UI캔버스에 스크립트를 넣고 스프라이트 아틀라스, 슬롯 프리팹을 가지게 해서
게임이 실행되면 인포매니져에서 샵데이터 딕셔너리의 길이만큼 반복문을 돌면서
슬롯 프리팹을 인스턴스화한다.
그 다음, 슬롯 오브젝트가 갖고있는 스크립트를 참조하여
슬롯의 정보를 표기하는 함수를 호출하며 그 슬롯의 id와 스프라이트를 매개변수로 넘겨준다.
슬롯의 id를 넘겨받은 슬롯 오브젝트는 자신이 갖고 있는 UI 요소에 
자신의 id에 해당하는 샵데이터 딕셔너리의 요소를 참조하여 정보를 표기한다.
